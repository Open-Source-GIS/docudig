About

DocuDig is a general purpose web application for fast, scalable searching, browsing, and serving of structured document data. It is written with Django and Solr on the backend. DocuDig began its life as a search application for WikiLeaks war log data, but has evolved to support arbitrary structured data.

Features

  • Faceted browsing, including date and numeric ranges
  • Full text searching
  • Expansion of acronyms
  • Free, Libre, Open Source Software

For downloads, installations, and usage, please see the project page at GitHub.

Use cases

If your data is uniformly structured (e.g., you could reasonably express your data set as a CSV file with a reasonable number of columns), large (thousands to hundreds of thousands of entries), contains acronyms that need to be identified and defined, and you need to be able to host on your own servers, support high load, to allow rich searching and browsing, and to expose an API for other developers to access your data, DocuDig might be for you.

You should be comfortable with:

  1. Correcting any inconsistencies in your document's structure, and formatting it into a particular CSV format.
  2. Defining your data set's structure in a configuration file.
  3. Editing a javascript file to define acronyms.
  4. Editing Django templates to customize the appearance of search excerpts and document pages.
  5. Deploying a Django application, and running Solr.
  6. For high load, deploying a caching server/HTTP accelerator like varnish.

Full instructions are available at the GitHub page.
